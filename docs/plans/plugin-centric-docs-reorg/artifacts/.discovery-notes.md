## Link topology — the crux of the implementation (D16, F3)

Each moved doc carries a dense web of relative links that must be recomputed for its new tree location. Categories
observed in sample docs (`docs/skills/han-coding/code-review.md`, `docs/agents/han-core/project-manager.md`):

| Link target category                          | Old form (from `docs/skills/{plugin}/x.md`) | New form (from `{plugin}/docs/skills/x.md`)            | Changes? |
| --------------------------------------------- | ------------------------------------------- | ----------------------------------------------------- | -------- |
| Repo-root `README.md`                         | `../../../README.md`                        | `../../../README.md`                                  | No (both 3 deep) |
| Same-plugin sibling long-form doc             | `./code-overview.md`                        | `./code-overview.md`                                  | No       |
| Other-plugin skill doc                        | `../han-github/post-code-review-to-pr.md`   | `../../../han-github/docs/skills/post-code-review-to-pr.md` | Yes |
| Agent doc (from a skill doc)                  | `../../agents/han-core/junior-developer.md` | `../../../han-core/docs/agents/junior-developer.md`   | Yes      |
| Top-level `docs/` page (yagni/sizing/evidence)| `../../yagni.md`                            | `../../../docs/yagni.md`                              | Yes      |
| Own index (`docs/skills/README.md`)           | `../README.md`                             | `../../../docs/skills/README.md`                     | Yes      |
| Cross index (`docs/agents/README.md`)         | `../../agents/README.md`                    | `../../../docs/agents/README.md`                     | Yes      |
| Own plugin `SKILL.md` / agent definition      | `../../../han-coding/skills/x/SKILL.md`     | `../../skills/x/SKILL.md`                             | Yes (shorter) |
| Own-plugin `references/` file                 | `../../../han-coding/references/yagni-rule.md` | `../../references/yagni-rule.md`                   | Yes (shorter) |
| Guidance references (han-plugin-builder)      | `../../../han-plugin-builder/skills/guidance/references/...` | `../../../han-plugin-builder/skills/guidance/references/...` | No |

Recomputation is per-target-category and depends on BOTH the doc's new location and whether the target also moved. A
naive `git mv` with a path find-and-replace on the doc's own path (rejected alternative in D16) misses every outbound
relative link. D14 also changes the "links up" first bullet: it must now point to the adjacent plugin README, then the
root.
