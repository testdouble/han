# Discovery Notes: han-core Restructure Implementation Planning

Single source of truth for project context during implementation planning. Specialists: read this first; do not
re-grep for anything recorded here. Search further only for what your domain needs that this file does not cover.

## Tech stack

- Pure-markdown Claude Code plugin monorepo. No build system, no test runner, no compiled code. Entities are
  `SKILL.md` files (skills), `.md` files with frontmatter (agents), and JSON manifests.
- Two manifest platforms: `.claude-plugin/plugin.json` (all ten plugins) and `.codex-plugin/plugin.json` (nine of
  ten plugins — all except han-linear).
- Suite catalog: `.claude-plugin/marketplace.json` at repo root. It lists all ten plugins with description prose and
  versions; it carries no dependency data of its own (dependencies live only in each plugin.json).
- Verification is manual/scripted inspection: link checking, grep sweeps, and real installs (per spec OI-1).

## Key mechanics from the investigation (all evidence IDs refer to docs/plans/han-core-restructure/investigation.md)

- Moved long-form docs carry ~19 relative `](../agents/...)` links into agents that stay in han-core; each must be
  rewritten to the cross-plugin form `../../../han-core/docs/agents/{name}.md` (V2; per-file counts:
  project-documentation.md 3, architectural-decision-record.md 5, runbook.md 5, research.md 2 staying + 1 moving,
  gap-analysis.md 6).
- The single cross-plugin Skill invocation into han-core is
  `han-atlassian/skills/project-documentation-to-confluence/SKILL.md:50` (`han-core:project-documentation`) (E7).
- han-core's `.codex-plugin/plugin.json` names moving skills in its longDescription, keywords ("documentation"),
  and defaultPrompt entries (V3).
- CLAUDE.md's Conventions section pins agents to `han-core/agents/` and `han-communication/agents/`; han-research
  becomes a third agent-owning plugin (V4).
- Rule files are vendored per-plugin, never read cross-plugin (E12) — new plugins vendor their own copies.
- project-discovery output is consumed by filename probe (`find . -name "project-discovery.md"`) from 13 skills in
  3 plugins; no consumer references the plugin path (E9).
- han-reporting/README.md:20 falsely claims its skills dispatch shared han-core agents (E4) — corrected under D5.
